Ecosystem

Small surface, clear roles.

KoppaJS is easier to trust when each package owns an explicit boundary and the official website explains how those pieces connect.

Runtime

@koppajs/koppajs-core

Registers components, runs controller code, and keeps rendering inspectable.

Build

@koppajs/koppajs-vite-plugin

Turns `.kpa` files into modules without blurring the boundary between build and runtime.

Navigation

@koppajs/koppajs-router

Delivers route matching and active-link state with explicit browser contracts.

Philosophy

Opinionated about clarity, not ceremony.

KoppaJS is designed for people who would rather inspect the system than memorize framework folklore.

Readable runtime contracts

The runtime should be explainable to another engineer without hand-waving.

Deliberate scope

Official packages should stay focused instead of becoming a bag of convenience magic.

AI-recoverable intent

Repository contracts and architecture notes should survive the absence of tribal knowledge.

Support
